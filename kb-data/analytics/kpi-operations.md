# Operations KPIs

Tracks manufacturing and roaster operations using PBI MOT (Minimum Order Target) Measures. Covers MOT achievement rates (period and cumulative), variance between target and actual weight, Barista's Choice vs non-BC weight splits, and country-by-week allocation tracking.

## Data Notes

- **MOT Achievement %** measures how closely actual roaster orders meet the Minimum Order Target. Values below 100% indicate under-ordering.
- **Cumulative Achievement** smooths weekly variation by aggregating across the period.
- **BC/NonBC weight split**: `kpi_mot_ordered_weight_bc` + `kpi_mot_ordered_weight_nonbc` = `kpi_mot_ordered_weight`. Barista's Choice orders are algorithmically allocated; non-BC is customer-selected.
- **Allocated Total** is the MOT target weight set per country per week. **Ordered Weight Total** is what was actually ordered. Variance shows the gap.
- **% mix** fields show relative distribution: `pct_mix_actual_kg` is actual weight share; `pct_mix_mot` is target weight share. Divergence indicates allocation drift.

## PBI Measures

This doc draws from the **MOT Measures** folder in BeanzCoreDatabricks:

- % MOT Achievement (period, Cumulative)
- % MOT vs Actual Total KG Variance
- Actual % of Total KG
- MOT Variance
- Sum of MOT OrderedWeight (total, BC, NonBC)
- Total Allocated MOT / Ordered Weight (by Country, by Week)
- Total OrderedWeight
- % mix of Actual Total KG / MOT
