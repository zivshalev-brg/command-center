# Subscription KPIs

Tracks subscription lifecycle across all beanz.com markets using PBI Subscription Measures. Covers total, active, cancelled, and paused counts with year-over-year comparisons, Barista's Choice splits, churn rate calculation, and subscription duration/LTV metrics.

## Data Notes

- **Active Subscriptions** includes PAUSED subscriptions in PBI's definition. Use `kpi_active_subs - kpi_paused_subs` for strict active count.
- **Barista's Choice split**: `kpi_active_bc_subs` + `kpi_active_non_bc_subs` = `kpi_active_subs`. BC subscribers receive curated selections; non-BC choose their own.
- **Churn Rate** uses the formula: `Count Subs Lost in Period / Count Subs Start of Period`. The `<1KG` variant isolates small-volume churn.
- **Cancellation Count** differs from **Cancelled Subscriptions**: cancellation count is the event count within a period; cancelled subs is the cumulative all-time total.
- **Subscription LTV** is calculated at the subscription level, not customer level. See [[kpi-customers]] for customer-level LTV.

## PBI Measures

This doc draws from the **Subscription Measures** folder in BeanzCoreDatabricks:

- Count of All / Active / Cancelled / Paused / New Subscriptions (+ LY, % Change)
- Count of New Subscriptions (Converted to Paid)
- Count of Active Subscriptions - Barista's Choice / Non-Barista's Choice (+ LY, % Change)
- Count of Subscriptions - No Paid Orders
- Cancellation Count - Subscriptions
- Count of All Subscriptions with Cancellation Promotion
- Subscription Churn (+ LY, % Change, <1KG variant)
- Subscription Churn - Count Subs Lost/Start of Period
- Average Sub Length in Days, Avg Subscription Length by Order Type
- Subscription Lifetime Value
